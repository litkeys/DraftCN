## Story

**As a** user,

**so that** I can refine my design.

## Acceptance Criteria

1. Blocks have no visible border by default
2. Click on block to select it
3. Clicking on one block deselects all other blocks
4. Click on canvas deselects all blocks
5. Selected block shows blue border
6. Blue border persists when block is selected
7. Hovering over a block shows blue border (same style as selected)
8. Backspace or Delete key removes selected block
9. Deleted block removed from state and canvas updates

### Selection State Architecture

**Unified BlocksSlice with Selection** [Source: architecture/data-models.md#Selection Architecture Note]:
The selection is managed within BlocksSlice using a two-tier approach:

- `selectedBlockIds: string[]` - Array serving as the single source of truth for selection
- `block.selected: boolean` - Synchronized field on each block for efficient rendering

When selection changes:

1. Update the `selectedBlockIds` array
2. Synchronize the `selected` field on affected blocks
3. Blocks render based on their own `selected` field

**Extended BlocksSlice Structure**:

```typescript
interface BlocksState {
  blocks: Block[]
  selectedBlockIds: string[] // Source of truth for selection
}

interface BlocksActions {
  // Existing actions
  addBlock: (block: Block) => void
  updateBlock: (id: string, updates: Partial<Block>) => void
  removeBlock: (id: string) => void

  // New selection actions
  selectBlock: (blockId: string) => void // Single selection (clears others)
  clearSelection: () => void // Deselect all blocks

  // Helper to sync block.selected with selectedBlockIds
  syncBlockSelection: () => void
}
```

For this story, we implement:

- `selectBlock` - Replaces current selection with single block
- `clearSelection` - Clears all selections
- Synchronization logic to update block.selected fields

**Implementation Example**:

````typescript
// In BlocksSlice
selectBlock: (blockId: string) => set((state) => {
  // Update selectedBlockIds array (source of truth)
  const newSelectedIds = [blockId];

  // Sync block.selected fields
  const updatedBlocks = state.blocks.map(block => ({
    ...block,
    selected: block.id === blockId
  }));

  return {
    selectedBlockIds: newSelectedIds,
    blocks: updatedBlocks
  };
}),

clearSelection: () => set((state) => ({
  selectedBlockIds: [],
  blocks: state.blocks.map(block => ({
    ...block,
    selected: false
  }))
}))

**Selection Selectors**:
```typescript
export const blocksSelectors = {
  // Existing selectors
  getAllBlocks: (state) => state.blocks,
  getBlockById: (state, id) => state.blocks.find(b => b.id === id),

  // New selection selectors
  getSelectedBlockIds: (state) => state.selectedBlockIds,
  hasSelection: (state) => state.selectedBlockIds.length > 0,
  isBlockSelected: (state, blockId) => state.selectedBlockIds.includes(blockId)
}
````

**Required Visual States** [Source: architecture/components.md#Block Rendering]:

- Hover (not selected): Blue border (border-blue-500) when block.selected === false

**Rendering Implementation**:

```typescript
// In Canvas component block rendering
{
  blocks.map((block) => {
    // ... existing template logic ...
    return (
      <div
        key={block.id}
        className={`absolute rounded cursor-pointer transition-colors
        ${
          block.selected
            ? 'border-2 border-blue-500'
            : 'border-2 border-transparent hover:border-blue-500'
        }`}
        style={{
          left: block.x,
          top: block.y,
          width: block.width,
          height: block.height,
          zIndex: block.z,
        }}
        onClick={(e) => handleBlockClick(block.id, e)}
        data-block-id={block.id}
        data-selected={block.selected}
        data-testid={`block-${block.id}`}
      >
        <Component {...block.props} />
      </div>
    )
  })
}
```

Note: Each block checks its own `selected` field for rendering, which is kept in sync by BlocksSlice

### Event Handling Patterns

**Click Event Handling**:

```typescript
const handleBlockClick = useCallback(
  (blockId: string, e: React.MouseEvent) => {
    e.stopPropagation() // Prevent canvas click
    selectBlock(blockId) // BlocksSlice handles deselection of others and sync
  },
  [selectBlock]
)

const handleCanvasClick = useCallback(
  (e: React.MouseEvent) => {
    if (e.target === e.currentTarget) {
      // Only if clicking canvas itself
      clearSelection() // BlocksSlice clears selectedBlockIds and syncs all blocks
    }
  },
  [clearSelection]
)
```

**Keyboard Event Handling** [Source: architecture/frontend-architecture.md#Canvas Direct Rendering Pattern]:

```typescript
// In useKeyboard hook
useEffect(() => {
  const handleKeyDown = (e: KeyboardEvent) => {
    if (e.key === 'Delete' || e.key === 'Backspace') {
      const selectedIds = useAppStore.getState().selectedBlockIds
      selectedIds.forEach((id) => {
        useAppStore.getState().removeBlock(id)
      })
      useAppStore.getState().clearSelection()
    }
  }

  window.addEventListener('keydown', handleKeyDown)
  return () => window.removeEventListener('keydown', handleKeyDown)
}, [])
```
