# Story 1.8: Block Movement on Canvas

## Status

## Story

**As a** user,
**I want** to reposition blocks after placing them,
**so that** I can adjust my layout.

## Acceptance Criteria

1. Blocks can be dragged to new position
2. Blocks are automatically selected on drag start
3. Block position updates on drag
4. No previews or ghosts during drag
5. State updates with new position after move
6. Only one block can be dragged at a time

**Solution**: Modify DropPreview to only render for library drags:

```typescript
  return null // Don't render preview for canvas block dragging

```

This ensures that when dragging existing blocks on the canvas, only the actual block moves (direct manipulation), with no ghost or preview element.

### Event Handler Interaction (MouseDown vs Click)

**Concern**: How `handleBlockMouseDown` works with existing `handleBlockClick` without conflicts.

**Solution**: The handlers naturally coexist:

1. **Quick Click**: mouseDown → mouseUp → click (block already selected from mouseDown, click is no-op)
2. **Drag**: mouseDown → mouseMove → mouseUp (browser doesn't fire click after drag)
3. **Safety Check**: `handleBlockClick` should check `if (dragManager.isDragging()) return;`

The browser naturally prevents click events after drag operations, and our `isDragging()` check provides additional safety. Both handlers use `stopPropagation()` to prevent canvas deselection.

### Block Drag Implementation Pattern

**Mouse Event Flow for Canvas Block Dragging**:

1. **MouseDown on Block**:

   - Always call `selectBlock(blockId)` to ensure only this block is selected
   - Start drag with `dragManager.startDrag('canvas', block, mousePosition)`
   - Calculate and store mouse offset from block origin

2. **MouseMove on Canvas**:

   - If dragging and sourceType is 'canvas'
   - Calculate new position: `newX = e.clientX - offset.x`, `newY = e.clientY - offset.y`
   - Update block position: `updateBlock(draggedItem.id, { x: newX, y: newY })`

3. **MouseUp on Canvas** (routing between drag types):

   ```typescript
   if (dragManager.isDragging()) {
     const { sourceType } = dragManager.getDragState()

     if (sourceType === 'canvas') {
       // Canvas block drag - just end the drag
       dragManager.endDrag()
     } else if (sourceType === 'library') {
       // Library drop - existing logic to create new block
       // ... existing handleDrop logic ...
     }
   }
   ```

**Implementation Example**:

```typescript
const handleBlockMouseDown = useCallback(
  (blockId: string, e: React.MouseEvent) => {
    e.stopPropagation() // Prevent canvas selection

    // Check if already dragging
    if (dragManager.isDragging()) return

    // Get the block
    const block = blocks.find((b) => b.id === blockId)
    if (!block) return

    // Always select only this block (clears multi-selection)
    selectBlock(blockId)

    // Get canvas bounds for coordinate system conversion
    if (!canvasRef.current) return
    const rect = canvasRef.current.getBoundingClientRect()

    // Convert mouse position to canvas-relative coordinates
    const canvasMouseX = e.clientX - rect.left
    const canvasMouseY = e.clientY - rect.top

    // Calculate offset from mouse to block origin (both in canvas coordinates)
    const offset = {
      x: canvasMouseX - block.x,
      y: canvasMouseY - block.y,
    }

    // Start drag
    dragManager.startDrag('canvas', block, { x: e.clientX, y: e.clientY })

    // Store offset in drag state for accurate dragging
    useAppStore.getState().setDragState({ offset })
  },
  [blocks, selectBlock]
)
```
