# Story 1.1: Project Setup and Basic Layout

## Status
Draft

## Story
**As a** developer,  
**I want** to initialize the Next.js project with proper configuration,  
**so that** I have a working foundation with all required dependencies.

## Acceptance Criteria
1. Next.js 15+ project created with App Router structure
2. TypeScript configured with strict mode enabled
3. Tailwind CSS installed and configured for styling
4. shadcn/ui initialized with base components
5. Zustand installed for state management
6. Basic layout with logo in top-left, left sidebar (20% width), and main canvas area (80% width) renders correctly
7. Project runs locally with npm run dev without errors

### Coding Standards
**[Source: architecture/coding-standards.md]**
- **Type Safety**: All code must be TypeScript with strict mode enabled
- **Component Patterns**: Use functional components with hooks, no class components
- **Naming Conventions**:
  - Components: PascalCase (e.g., `BlockInstance.tsx`)
  - Hooks: camelCase with 'use' prefix (e.g., `useDrag.ts`)
  - Functions: camelCase (e.g., `snapToGrid()`)
  - Types/Interfaces: PascalCase (e.g., `BlockTemplate`)
  - CSS Classes: kebab-case (e.g., `canvas-grid`)
  - Constants: UPPER_SNAKE_CASE (e.g., `GRID_SIZE`)

### Setup Instructions from Latest Docs
**[Source: context7 - Next.js & shadcn/ui latest]**
1. **Next.js 15 Setup**: Use `npx create-next-app@latest` for initialization
2. **shadcn/ui Setup**: Use `npx shadcn@latest init` after Next.js setup
3. **Component Installation**: Use `npx shadcn@latest add [component]` for each component
4. **Zustand Installation**: Use npm to install exact version `zustand@^4.5`

### Testing
**[Source: architecture/testing-strategy.md]**
- **Test Framework**: Vitest 1.0+ for unit testing (to be set up in future stories)
- **Test Location**: tests/ directory at project root
- **Test File Naming**: *.test.ts or *.test.tsx
- **Component Tests**: Should verify rendering and basic interactions
- **No backend testing needed for MVP** (client-side only)
