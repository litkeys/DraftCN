# Story 1.6: Freeform Drag and Drop with Template Instantiation

## Status

Approved

## Story

**As a** user,  
**I want** to drag blocks from the library to the canvas,  
**so that** I can build my layout visually.

## Acceptance Criteria

1. Templates draggable from sidebar using mouse
2. Dragged template preview follows cursor smoothly during drag
3. Template can be dropped anywhere on canvas (freeform, no grid)
4. On drop, block instance created from template with defaultProps
5. Block instance gets unique ID and references template typeId
6. New blocks get sequential z-index (1, 2, 3, etc.)
7. Drag operation can be cancelled with Escape key

### Drag Manager Architecture

**Drag Manager Responsibilities** [Source: architecture/components.md#Drag Manager]:
The Drag Manager manages drag state and coordinates drag lifecycle. Follows separation of concerns principle where DragManager handles state management while Canvas handles drop detection and positioning:

- `startDrag(source: 'library' | 'canvas', item: any)` - Begin drag operation
- `updateDragPosition(x: number, y: number)` - Track mouse movement
- `endDrag()` - Complete drag operation (Canvas determines drop position)
- `cancelDrag()` - Cancel current drag

**Separation of Concerns**:
- **DragManager:** Manages drag state, handles Escape key cancellation, notifies store of state changes
- **Canvas:** Detects drop events, calculates drop positions, validates drop zones, creates block instances

**Technology Stack** [Source: architecture/components.md#Drag Manager]:

- React 19 event handlers
- Zustand for drag state management

### State Management Structure

**Drag State Interface** [Source: architecture/frontend-architecture.md#State Structure]:

```typescript
drag: {
  isActive: boolean

  draggedItem: any
```

- `endDrag: () => void`

### Core Workflow - Add Block from Library

**Drag and Drop Sequence** [Source: architecture/core-workflows.md#Add Block from Library to Canvas]:

1. User mousedown on template thumbnail → Sidebar gets template from registry
2. Sidebar calls `DragManager.startDrag('library', template)`
3. DragManager updates store with `setDragState({active: true, sourceType: 'library'})`
4. During drag, Canvas shows drop preview following cursor
5. On mouseup, Canvas creates block instance via `registry.generateBlockInstance(typeId)`
6. Canvas updates block position and adds to store with `addBlock(block)`
7. DragManager clears drag state

### Block Instance Creation

**Block Data Model** [Source: architecture/data-models.md#Block]:

```typescript
interface Block {
  id: string // Unique identifier
  typeId: string // References template
  props: any // From template.defaultProps
  x: number // Horizontal position in pixels
  y: number // Vertical position in pixels
  width: number // From template.defaultWidth
  height: number // From template.defaultHeight
  z: number // Sequential stacking order
  selected: boolean // Initially false
}
```

**Instance Creation Method** [Source: lib/blocks/registry.ts#generateBlockInstance]:

The registry's `generateBlockInstance` method creates a block with:

- Unique ID format: `${typeId}-${Date.now()}-${Math.random().toString(36).substring(2, 11)}`
- Props: Merges template.defaultProps with any overrideProps
- Initial position: x: 0, y: 0 (must be updated after creation)
- Dimensions: From template.defaultWidth and template.defaultHeight
- Z-index: Initially 0 (must be updated after creation)
- Returns null if template not found

Note: After calling `generateBlockInstance`, the Canvas must:

1. Update the block's x, y position to the drop coordinates
2. Calculate and set the correct z-index (sequential: 1, 2, 3...)
3. Add the block to the store
